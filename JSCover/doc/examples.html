The file system example can be run online by clicking on the links below. The server examples need to be downloaded and run manually. For all examples see the 'examples' directory of the ZIP distribution.

JSCover Server

Run jscover-server.sh (see below) and then go to http://localhost:8080/jscoverage.html?/index.html.

java -jar target/dist/JSCover.jar -ws --document-root=doc/example --report-dir=target

Manually run the command below the go to: http://localhost:8080/jscoverage.html?/test/index.html.

java -jar target/dist/JSCover.jar -ws --document-root=doc/example-qunit/src --report-dir=target

Now try again with the '--no-instrument=/test' option to remove the test scripts from the report.

java -jar target/dist/JSCover.jar -ws --document-root=doc/example-qunit/src --report-dir=target --no-instrument=/test

JSCover File Instrumentation

Run jscover-filesystem.sh (see below) and then go to QUnit test suite.

java -jar target/dist/JSCover.jar -fs --no-instrument=test doc/example-qunit/src doc/example-qunit/out

PhantomJS & JSCover Server

This example uses QUnit and run-jscover-qunit.js, but can easily be run with Jasmine. See the file run-jscover-jasmine.js and the manual for more details.

Download PhantomJS and put the executable on your path.

Run the web-server, jscover-server.sh

Run phantom-ws.sh

phantomjs src/test/javascript/lib/PhantomJS/run-jscover-qunit.js http://localhost:8080/test/index.html

The QUnit test results appear in the PhantomJS console, and the coverage report should be in 'target/phantom'. Don't forget to shutdown the server.
